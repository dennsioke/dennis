Title
Autoplay with per-slide duration and direction control

Many content-heavy pages - marketing sites, portfolio showcases, and e-commerce banners - use carousels to tell a story that passive visitors would otherwise miss. Right now `AwesomeSlider` requires manual navigation, so autoplay needs to be added.
Enabling it should require just an `autoplay` prop. The global dwell time is set by `autoplayInterval` (default 3000ms). Teams that need faster cycling can set `autoplaySpeed`; the effective dwell time is `interval / speed`, so a speed of 2 halves the wait. The result is rounded to the nearest integer millisecond (since `setTimeout` takes integer values) and then clamped to a minimum of 50ms. Invalid speed values - zero, negative, NaN, or Infinity - fall back to 1. Direction is controlled by `autoplayDirection`, accepting "next" (default) or "prev" for decks that play backwards.
Individual slides can declare their own `duration` in milliseconds to override the global interval. Zero, negative, and NaN values are ignored and fall back to `autoplayInterval`, so a bad data entry in one slide does not stall the whole deck. Infinity is accepted as a valid (positive) override but is non-finite, so after speed scaling it cannot be represented as a real timer value; it is normalized to the 50ms minimum rather than rejected outright.
The slider must not begin cycling until it has fully started; when a startup screen is configured autoplay waits until that phase completes. Once running, `pauseOnHover` pauses on mouse-enter and resumes on mouse-leave, unless autoplay was explicitly stopped via `pause()` - in that case moving the cursor away must not restart cycling. For programmatic control, `pause()` and `play()` methods are exposed on the component ref - useful for stopping the carousel while a modal is open. Calling `play()` when already playing is idempotent and must not stack duplicate timers. The `onTransitionStart` and `onTransitionEnd` callbacks fire on every navigation whether triggered by the user or by autoplay, so existing analytics hooks need no changes.
When `infinite` is false the slider stops at the terminal slide; when true (the default) it wraps in either direction. A `disabled` slider or one with a single slide suppresses autoplay entirely. Any manual arrow or bullet click cancels the pending timer and reschedules it using the incoming slide's effective dwell time.
