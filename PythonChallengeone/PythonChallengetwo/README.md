# Holiday Calendar Diff

Repository
https://github.com/vacanza/holidays
Commit: 9d862c4be35465faa23918f827d31b94a1a2e896

Users cannot compare two HolidayBase instances to identify what changed between them. A `calendar_diff` utility (accessible as `holidays.calendar_diff`) should categorize differences into four groups: holidays added in the second instance, holidays removed from the first, holidays where the name changed on the same date (renamed), and holidays that moved to a nearby date while keeping a matching name (shifted).
Dates present in both calendars are evaluated for renamed before shifted detection; a date that appears in both instances cannot participate in shifted detection regardless of nearby name matches. Dates with identical name strings in both calendars produce no entry in any category. Renamed detection compares the full name string; entries identified as renamed must not appear in added or removed. Shifted detection splits multi-name entries by the library's holiday name delimiter ("; ") and matches each component using a `name_lookup` strategy (default "icontains", same values accepted as `HolidayBase.get_named`); when multiple h2 candidates fall within the window the nearest date is chosen. Shift distance is measured in absolute days so a holiday moving to an earlier date is detected as shifted. Each h2 date can only participate in one shifted pairing; if multiple h1 dates match the same h2 date only one pairing is formed. In shifted entries old_name records the matched individual component, not the full delimiter-joined string. A `shifted_window` parameter (default 7) sets the maximum days a holiday may move and still count as shifted - a shift of exactly `shifted_window` days is classified as shifted, one day beyond is not. Setting `shifted_window=0` disables shifted detection; holidays identified as shifted must not appear in added or removed. The result must expose `added`, `removed`, `shifted`, and `renamed` as list attributes where `added`/`removed` entries are `(date, name)` tuples, `shifted` entries are `(old_date, old_name, new_date, new_name)`, and `renamed` entries are `(date, old_name, new_name)`.
